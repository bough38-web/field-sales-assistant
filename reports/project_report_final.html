[보고] 영업기회 발굴 시스템(Field Sales Assistant) 기능 개선 완료

수신: 관련 부서 제위

일자: 2026. 01. 22

제목: 사용자 편의성 증대 및 시스템 고도화 개발 완료 보고

1. 개요 (Overview)

본 문서는 '영업기회 발굴 어시스턴트' 앱의 사용자 경험(UX) 개선 및 운영 효율성을 높이기 위해 수행된 주요 업데이트 내역을 보고합니다. 금번 업데이트는 현장 영업 담당자의 사용 편의성과 데이터 보안 강화에 중점을 두었습니다.

2. 주요 개선 내역 (Key Improvements)

가. 사용자 매뉴얼 접근성 및 가독성 획기적 개선

  • 현황 및 문제점: 기존에는 로그인 화면 내 좁은 영역(Expander)에서 매뉴얼이 표시되어, 텍스트와 이미지가 작게 보이고 가독성이 현저히 저하됨.
  • 개선 내용:
    • '사용설명서 보기 (새 창)' 버튼을 도입하여 클릭 시 독립된 웹 페이지에서 매뉴얼을 전체 화면으로 열람 가능하도록 변경.
    • 전용 페이지(pages/user_manual.py)를 신설하여 모바일 및 PC 환경에 최적화된 반응형 레이아웃 적용.
  • 기대 효과: 신규 사용자의 앱 적응 시간 단축 및 매뉴얼 활용도 증대.

나. 랜딩 페이지 UX 최적화 (사용자 중심 설계)

  • 개선 내용: 앱 접속 시 노출되는 기본 로그인 탭을 기존 '관리자'에서 '담당자(Manager)'로 변경.
  • 사유: 실제 앱을 가장 빈번하게 사용하는 주체는 현장 영업 담당자이므로, 불필요한 탭 전환 단계를 제거하여 접근성을 높임.

다. 지도 시각화 기능 고도화 및 정보 보호

  • 지도 범례(Legend) 추가: 지도 상단에 직관적인 범례를 추가하여 데이터 식별력 강화.
    • 🔵 파랑: 영업/정상
    • 🔴 빨강: 폐업/정지
    • 🟣 보라: 100평 이상 대형 매장 (신규 강조 표시)
  • 개인정보 보호 조치: 지도 팝업(Tooltip)에 표시되는 담당자 실명을 마스킹 처리(예: 홍길동 → 홍*동)하여 개인정보 노출 최소화.

라. 보안성 강화

  • 관리자 계정: 관리자(Admin) 접속 패스워드의 복잡도를 상향 조정하여 보안성 강화 (admin1234 → admin1234!).

3. 버그 수정 및 안정화

배포 환경 경로 수정: 배포 서버(GitHub/Cloud) 환경에서 사용자 매뉴얼 파일 경로가 올바르게 인식되지 않는 문제를 수정(reports/ → static/ 경로 재설정)하여 안정적인 서비스를 보장함.

4. 향후 계획

  • 사용자 피드백을 수렴하여 검색 필터링 기능 강화 예정.
  • 모바일 환경에서의 지도 로딩 속도 추가 최적화 검토.
